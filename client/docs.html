Pokemon API Documentation

This API allows users to view, filter, add, and edit Pokemon data.

GET /pokemon

Returns all Pokemon.

Optional Query Parameters:

  • ?type=fire – filters Pokemon by type
  • ?name=Pikachu – filters by exact name (case-insensitive)

Example Request:

GET /pokemon?type=grass

Response (200):

[
  {
    "id": 1,
    "name": "Bulbasaur",
    "type": ["grass", "poison"],
    "img": "image-url"
  }
]

GET /pokemonById

Required Query Parameter:

  • ?id=25

Example:

GET /pokemonById?id=25

Success Response (200):

{
  "id": 25,
  "name": "Pikachu",
  "type": ["electric"],
  "img": "image-url"
}

Error (400):

{
  "message": "id parameter is required"
}

Error (404):

{
  "message": "Pokemon not found"
}

GET /types

Returns all unique Pokemon types.

Example:

GET /types

Response (200):

[
  "normal",
  "fire",
  "water",
  "grass"
]

GET /stats

Returns metadata about the dataset.

Example:

GET /stats

Response:

{
  "totalPokemon": 151
}

POST /addPokemon

Adds a new Pokemon.

Required Fields:

  • name (string)
  • type (array or string)

Optional:

  • img (string URL)

Example Request Body:

{
  "name": "Testmon",
  "type": ["fire", "dragon"],
  "img": "https://example.com/image.png"
}

Success (201):

{
  "id": 152,
  "name": "Testmon",
  "type": ["fire", "dragon"],
  "img": "https://example.com/image.png"
}

Error (400):

{
  "message": "name and type are required"
}

POST /editPokemon

Edits an existing Pokemon.

Required:

  • id

Optional Fields:

  • name
  • type
  • img

Example:

{
  "id": 25,
  "type": ["electric", "fighting"]
}

Success (204):

No response body.

Error (404):

{
  "message": "Pokemon not found"
}
